1. Поддерживаемые команды:

    * to (на) - перенаправление на компьютер локальной сети.
    * run (запустить) - запустить файл или папку. Например: run C:\ - запустит папку C:\
    * shutdown (выключить) - выключение компьютера
    * reboot (перезагрузить) - перезагрузить компьютер
    * shut (выйти) - завершение сеанса
    * msg (сообщение) - показывает сообщение. Например: to comp key msg Привет - выведет сообщение Привет на локальном компьютере comp, где запущена программа-сервер с ключом key.
    * define (мкр) - работа с макроопределениями (синонимы команд). Например: define del |mac| - удаляет макроопределение mac;  define |"mac 2"| |A B C D F| - добавит/заменит макроопределение (mac 2), которое будет заменяться командой (A) с параметром этой команды (B C D F).;define |mac_3| |"A B C D" F| - добавит/заменит макроопределение mac_3, которое будет заменяться командой (A B C D) с параметром этой команды (F).
    * stop (остановить) - остановить программу-сервер
    * port (порт) - сменить прослушиваемый порт и включить сетевые функции (прослушивание порта)
    * popen - вызов команды (консольного приложения) с выводом в программе-сервере результатов, распечатанных в окне этого консольного приложения. Например: popen tasklist - отобразит список всех запущенных процессов;
    * io - утановка/чтение портов компьютера. Например: io 888 - выведет значение порта 888; io  888  255 - установит порт 888  в значение 255; io help - справка по команде.
    * device - управляет портами принтера (LPT - разъем), используется как device  NUM  SET , где NUM - номер устройства (номер бита регистра data) и принимает значения от 1 до 8 , SET - включение / выключение устройства (принимает значения 0 - выключено, 1 - включено). Пример использования: device 1 - отобразит состояние устройства 1 (включено или выключено); devise 4 0 - выключит устройство 4; device help - отобразит справку по команде.
    * cmd (кмд) - поддержка любой команды из командной строки. Например: cmd netsh.exe wlan connect name=HOME - включит беспроводную сеть HOME.
    * выполнения любой команды, которая поддерживается в Пуск->выполнить. Например: regedit c:\1.reg - занесет в реестр данные из файла 1.reg.
    * стол - замена рисунка рабочего стола (popen "стол help" -вызывает справку). Например: стол с:\1.jpg - установка в качестве рисунка рабочего стола файла 1.jpg; стол_cnf.txt - сюда прописывается папка; стол - установка случайного рисунка из папки, заданной в стол_cnf.txt.
    * архиватор - шифрование файла. При вызове с одним параметром, этот параметр - путь к файлу, который нужно архивировать, файл зашивается в .ехе-файл с именем архивируемого файла, также создается файл-ключ с расширением .key и именем архивируемого файла. Чтобы открыть архив надо запустить этот .ехе-файл и указать пароль. Или запустить .ехе-файл с параметром, который указывает путь к файлу-ключу. При архивировании можно создать архив, который откроется только на этом компьютере, даже если злоумышленнику известен пароль, он его не откроет архив на другом компьютере.
    * +  : операция соединения строк. Строки соединять можно в 4-х форматах: "ab"+"cd" , "ab"+cd , ab+"cd" , ab+cd. Результат получается не в кавыках: abcd. Если требуется результат в кавычках задаем в одном из форматов: ""ab"+"cd"" , ""ab"+cd" , "ab+"cd"" , "ab+cd". Результат: "abcd"
    * {c++ ...} - компиляция и выполнение кода написанного на с++
    * file (файл) - выполнить команды записанные в файле. Например: file "C:\my_commands.txt", если файл находится в папке \command\ , то можно указывать только имя файла: file "my_com.txt"
    * disconnet (отклсеть) - отключить сетевые функции (прослушивание порта)


2. Формат вызова через командную строку:

	Ключ Логин Комманда

Пример(запуск коммандной строки):
	123 admin run cmd
Пример(запуск коммандной строки, без указания пользователя):
	123 NULL run cmd

3. Запуск в скрытом режиме
	
	/hide Логин Пароль

	В результате программа работает как обычно, но ее окно скрыто. Удобно, если программа запускается только для приема команд с другого компьютера.

4. В одной строке можно использовать несколько команд, указав их через точку с запятой (;)
5. Кавычки (") и символ (|) имеют одинаковое назначение, но в строке заключенной в символы (|) не происходит подстановка макропопределений. 
6. Возможно использование функций, которые заключаются в круглые скобки (...). Функции выполняются раньше, чем остальная часть комманды. Возможно использование ложенных функций.
7. В качестве путей до файла компилятора с++ можно использовать лишь пути не содержащие пробелов
8. Щелчек по в нижней области окна раскрывает область настроек. Чтобы скрыть снова щелкните по окну.
